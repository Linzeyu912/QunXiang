# 群像 QunXiang

**从小说文本中提取角色、场景、道具与世界观设定，扩写为生成式提示词，产出可复用的数字资产。**

*Turn novel text into structured entities — characters, scenes, props, and worldbuilding — then expand them into generation-ready prompts for digital IP assets.*

---

## 这是什么

「群像」是一个开源的**小说/剧本实体抽取与数字资产生成引擎**。

短剧、动画、游戏、AI 内容创作的上游都是文本，但一段小说原文没法直接喂给图像/视频生成模型——角色长什么样、场景是什么氛围、道具有什么细节，都散落在成百上千段描述里，且分散在几十章节中反复出现、彼此矛盾或互相补充。

群像做的事情，是把这堆散落的文字，整理成一份可以被创作者、生成模型、资产库直接使用的结构化资料：

```
小说 / 剧本 / 设定集文本
        │
        ▼
   实体抽取（角色 / 场景 / 道具 / 设定）
        │
        ▼
   描述聚合 + 原文索引（跨章节合并、出处可追溯）
        │
        ▼
   描述扩写（补全细节、消除歧义、风格化）
        │
        ▼
   生成式提示词（图像 / 视频 / 3D 模型等下游用途）
        │
        ▼
   结构化数字资产（可导出、可复用、可共建）
```

产出的资产最终服务于短剧制作流程（选角、置景、道具设计、分镜）和数字 IP 社区（创作者围绕同一部作品的角色/场景库进行二创、扩展、共建）。

## 核心能力

- **多类型实体抽取**——角色（姓名、别名、外貌、性格）、场景（地点、氛围、时间）、道具（外观、归属、作用）、设定（世界观规则、势力、历史背景）
- **跨章节实体消歧与合并**——同一角色在不同章节的描述自动关联、去重、补全，而不是抽取出一堆碎片
- **原文索引**——每条描述都保留对原文位置（章节/段落/引用片段）的追溯，方便核对与人工修订
- **描述扩写**——将小说中零散、克制的原文描述，扩写为细节完整、适合下游生成模型理解的结构化描述
- **提示词生成**——针对图像/视频生成场景，将扩写后的描述转换为可直接使用的生成式提示词（支持多种提示词风格/模型偏好）
- **可插拔 LLM 后端**——本地模型、云端 API、Mock 模式均可切换，方便按成本和精度取舍
- **人工审核环节**——自动抽取结果保留人工确认/修正的空间，而不是全自动黑箱产出
- **多格式导出**——JSON / Markdown / CSV，方便接入下游生产工具链或社区平台

## 应用场景

- **短剧制作**：开机前快速产出角色小传、场景设定集、道具清单，供选角、美术、置景团队直接使用
- **数字 IP 社区**：围绕一部小说/剧本生成可共享的角色卡、场景卡，创作者可以在统一资产库基础上进行二创、拓展支线、共建世界观
- **AI 内容生产流水线**：作为文本到视觉生成的中间层，产出结构化提示词供图像/视频生成模型使用

## 测试语料说明

开发过程中使用了若干公开网络小说（如《斗破苍穹》《凡人修仙传》等）作为测试语料，用来验证抽取流水线在长篇、多章节、人物关系复杂场景下的效果。`output/` 目录保留了部分真实跑出来的处理结果（角色/场景/道具的结构化描述、生成提示词、故事资产包等），供参考流水线的产出质量。

**仓库不包含任何小说原文**——原始文本受版权保护，不适合公开分发。抽取过程中用于溯源的原文引用片段（如 `sourceDescription`、`evidenceSnippets` 中的原文语句）在发布前已从 `output/` 中清除，只保留经过归纳、扩写后的处理结果和章节/段落等定位信息。如需重新生成完整效果（含原文引用），请自行准备已获授权的文本作为输入本地运行。

流水线的产出是结构化 JSON，大致形态如下（示例为占位内容，非真实抽取结果）：

```json
{
  "entityType": "character",
  "name": "示例角色名",
  "aliases": ["别名一", "别名二"],
  "description": "扩写后的角色描述……",
  "sourceRefs": [
    { "chapter": 3, "paragraph": 12 }
  ],
  "confidence": 0.92
}
```

`sourceRefs` 只记录章节/段落等定位信息，不保留原文内容。
